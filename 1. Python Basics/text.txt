📌 Lists
Definition: Ordered, mutable collections that allow duplicates.

Creation: Using square brackets [] or list() constructor.

Data types: Can hold mixed types (numbers, strings, booleans, other lists).

Accessing items: Indexing (positive & negative), slicing with start:end:step.

Mutability: Items can be changed, added (append, insert), or removed (remove, pop).

Operations: Membership check (in), sorting (sort, sorted), reversing (reverse).

Combining lists: Concatenation (+), extending (extend).

Traversing: Iteration with for, or enumerate for index + value.

List comprehension: Compact way to build lists with conditions/expressions.

Pitfall: Multiplying inner lists ([[0]] * 5) creates shared references; use comprehension for independent copies.

Other methods: Standard list utilities like count, index, etc.

📌 Tuples
Definition: Ordered, immutable collections (a.k.a. immutable lists).

Creation: Round brackets (), single-element tuples need a trailing comma.

Unpacking: Assign tuple elements to variables directly.

Similarity to lists: Support indexing, slicing, iteration.

Why tuples?

Immutable (data safety).

Faster and memory-efficient.

Hashable (usable as dictionary keys or set elements).

📌 Sets
Definition: Unordered, unindexed collections of unique, immutable elements.

Creation: Curly braces {} or set() constructor.

Valid types: Immutable types (int, float, str, tuple, bool). Lists are invalid.

Mechanism: Uses hashing for fast membership testing and uniqueness.

Access: No indexing; use in operator or iteration.

Modification: add, update, union, remove, discard, pop, clear.

Operators: Union (|), intersection (&), difference (-), symmetric difference (^), subset/superset (<=, >=), equality (==).

Invalid operators: Concatenation, multiplication, indexing, slicing.

Set comprehension: {expression for item in iterable if condition}.

Frozenset: Immutable, hashable version of a set.

📌 Dictionaries
Definition: Ordered (since Python 3.7), mutable collections of key–value pairs.

Creation: Curly braces {}, dict() constructor, or comprehension.

Keys: Must be immutable (int, float, str, tuple, bool). Lists are invalid.

Operations:

Access: dict[key] or safer dict.get(key, default).

Add/Update: dict[key] = value.

Remove: pop, del.

Membership: key in dict.

Iteration:

Over keys: for key in dict.

Over values: for value in dict.values().

Over pairs: for key, value in dict.items().

Copying:

Assignment (=): Both variables reference the same dictionary.

Shallow copy (copy(), dict(), {**dict}): New dictionary, but nested mutable objects are shared.

Deep copy (copy.deepcopy()): Fully independent copy, including nested objects.

Advantages: Efficient retrieval, fast membership testing, dynamic size.

Pitfalls:

KeyError when accessing non-existent keys (use .get()).

Keys must be immutable.
